# Maze Generator and Solver

Консольное приложение для генерации лабиринтов и поиска пути в них.

## Возможности

* **Генерация лабиринтов:** DFS (поиск в глубину) и алгоритм Прима
* **Решение лабиринтов:** A* и алгоритм Дейкстры
* **Гибкий ввод/вывод:** работа с файлами, stdin/stdout, или пайпами
* **Визуализация:** текстовое представление лабиринта с отмеченным путем

## Быстрый старт

```bash
# Сгенерировать лабиринт 10x10 и сохранить в файл
go run ./cmd/maze generate --width=10 --height=10 --output=maze.txt

# Решить лабиринт из файла
go run ./cmd/maze solve --file=maze.txt --start=0,0 --end=9,9 --output=solution.txt

# Сгенерировать и сразу решить (через pipe)
go run ./cmd/maze generate --width=10 --height=10 | \
  go run ./cmd/maze solve --start=0,0 --end=9,9
```

## Команды

#### generate - Генерация лабиринта

```bash
go run ./cmd/maze generate [флаги]
```

**Флаги**:

* **--algorithm** - алгоритм генерации: dfs (по умолчанию), prim
* **--width** - ширина лабиринта (обязательно)
* **--height** - высота лабиринта (обязательно)
* **--output** - путь к файлу для сохранения (если не указан - вывод в stdout)

Примеры:
```bash
# Генерация с алгоритмом DFS
go run ./cmd/maze generate --width=15 --height=15 --output=maze.txt

# Генерация с алгоритмом Прима, вывод в консоль
go run ./cmd/maze generate --algorithm=prim --width=20 --height=20
```

---

#### solve - Решение лабиринта;

```bash
go run ./cmd/maze solve [флаги]
```

**Флаги**:

* **--algorithm** - алгоритм поиска пути: astar (по умолчанию), dijkstra
* **--file** - путь к файлу с лабиринтом (используйте - для чтения из stdin)
* **--start** - начальная точка в формате x,y (обязательно)
* **--end** - конечная точка в формате x,y (обязательно)
* **--output** - путь к файлу для сохранения решения (если не указан - вывод в stdout)

````bash
# Решение A*
go run ./cmd/maze solve --file=maze.txt --start=0,0 --end=9,9 --output=solution.txt

# Решение Дейкстрой, вывод в консоль
go run ./cmd/maze solve --algorithm=dijkstra --file=maze.txt --start=0,0 --end=9,9

# Чтение из stdin
cat maze.txt | go run ./cmd/maze solve --start=0,0 --end=9,9
```
````

## Формат файла
**Обозначения:**
- `#` - стена (непроходимая ячейка)
- ` ` (пробел) - проход (проходимая ячейка)
- `O` - начальная точка (только в решении)
- `X` - конечная точка (только в решении)
- `.` - путь решения (только в решении)

**Пример лабиринта:**

```
###########
#O#   #  .#
#.### # #.#
#...#   #.#
###.##### #
#...#   #X#
###########
```

## Использование с Make

```bash	
# Показать все доступные команды
make help

# Сгенерировать лабиринт 10x10 в ./output/maze.txt
make generate

# Решить лабиринт из ./output/maze.txt и сохранить в ./output/maze_solution.txt
make solve

# Сгенерировать и сразу решить (pipe)
make generate-and-solve

# Полная демонстрация с промежуточным выводом
make demo

# Собрать бинарный файл
make build

# Запустить тесты
make test

# Очистить сгенерированные файлы
make clean
```
